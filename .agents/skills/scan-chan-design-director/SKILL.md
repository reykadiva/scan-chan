# Scan Chan Design Director

Use this skill as the mandatory creative review layer for Scan Chan. It evaluates visual, UX, interaction, motion, mascot, brand, and emotional decisions before implementation.

This skill never replaces project documentation. It routes decisions to the correct source documents and rejects work that conflicts with them.

## Required Review Flow

1. Apply `scan-chan-core-rules` first.
2. Apply `scan-chan-development-workflow` for implementation sequencing.
3. Read `references/design-document-router.md` to choose the governing docs.
4. Read the relevant checklist in `references/creative-review-checklists.md`.
5. Approve, revise, or reject the design decision before implementation.

## Approval Standard

Approve only when the decision:

- Aligns with all governing source documents.
- Keeps the pet emotionally central when the surface involves gameplay or home/scanner experience.
- Feels specific to Scan Chan rather than generic mobile, SaaS, or AI-generated UI.
- Uses documented design tokens, typography, color, motion, mascot behavior, and brand voice.
- Preserves warmth, calm, emotional honesty, accessibility, and visual clarity.

Reject or redesign when any governing document is violated. If documents conflict, stop and explain the conflict instead of guessing.

## Review Outcomes

Use one of these outcomes:

- **Approved**: Ready for implementation.
- **Approved with constraints**: Implementation may proceed only with stated constraints.
- **Revise**: Direction is close but must change before implementation.
- **Rejected**: Direction contradicts the source documents or Scan Chan identity.

Always name the documents and checklist areas used in the review.
