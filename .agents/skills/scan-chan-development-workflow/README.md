# Scan Chan Development Workflow

Permanent workspace skill for how AI contributors should implement Scan Chan changes.

This skill defines the required engineering workflow before, during, and after implementation. It is not the gameplay bible and not the architecture source of truth. It extends `scan-chan-core-rules` and points contributors to the authoritative project documents.

## Contents

- `SKILL.md` - required workflow sequence.
- `references/planning-gate.md` - pre-coding planning and documentation-read gate.
- `references/implementation-policies.md` - reuse, state, files, naming, tokens, animation, accessibility, and performance policies.
- `references/review-and-completion.md` - self-review, documentation, changelog, PR readiness, and completion checklists.
- `agents/openai.yaml` - Codex skill metadata.

## Maintenance

Keep this skill procedural. Put project rules and product truth in `docs/` or `scan-chan-core-rules`; put repeatable workflow steps here.
