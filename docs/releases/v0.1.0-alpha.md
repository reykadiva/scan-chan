# Scan Chan v0.1.0-alpha: First Companion

**Release Date**: July 5, 2026  
**Version**: 0.1.0-alpha  
**Codename**: First Companion  
**Release Type**: Alpha (Guest Mode Only)

---

## Highlights

First alpha release of Scan Chan, a virtual pet companion experience. This release establishes the foundation for pet care through barcode scanning, stat tracking, missions, achievements, and inventory management.

**Guest Mode Ready**: Full localStorage-based persistence without authentication required.

---

## New Features

### Core Systems
- **Pet System**: Five core stats (Hunger, Mood, Energy, Affection, Curiosity) with non-punitive decay
- **Home Hub**: Pet summary, daily status cards, greeting system, stat visualization
- **Scanner Foundation**: Scanner page with product lookup pipeline (camera integration incomplete)
- **Inventory System**: Collection management with stackable items, filtering, sorting, search
- **Mission System**: Daily and weekly missions with progress tracking
- **Achievement System**: 13 base achievements with progress tracking and unlock notifications
- **XP & Progression**: Level system with cumulative XP requirements

### Accessibility
- **Keyboard Navigation**: Skip navigation, focus management, tab order
- **Reduced Motion**: OS-level preference support and manual toggle
- **Color Contrast**: WCAG AA compliance (4.5:1 text, 3:1 interactive)
- **Touch Targets**: 44×44px minimum with proper spacing
- **Screen Reader Support**: ARIA labels, landmarks, semantic HTML
- **Responsive Design**: 320px-1280px+ breakpoints validated

### Performance
- **Lazy Loading**: Inventory client dynamically imported (24 KB bundle reduction)
- **Product Lookup Cache**: Map-based cache for repeated scans (495ms savings)
- **Narrow Selectors**: Zustand store selectors optimized to reduce re-renders

## Known Limitations

### Incomplete Features
1. **Camera Integration**: Scanner page implemented, camera hardware access not wired (placeholder UI present)
2. **Pet Rendering**: Mascot runtime foundation complete, pixel sprite rendering not implemented (placeholder present)
3. **Arashu Mode**: Server sync infrastructure ready, authentication not implemented
4. **Settings Page**: Settings store functional, dedicated settings UI not implemented

### Documented Issues
- Build hydration warnings (cosmetic, no runtime impact)
- Legacy component `<img>` usage (5 warnings, acceptable for legacy quarantine)
- Diagnostic scripts lint errors (test utilities only)

## Architecture

- **Framework**: Next.js 16 with React 19
- **State**: Zustand with localStorage persistence
- **Database**: Dual Prisma clients (Guest/Arashu isolation)
- **Styling**: Tailwind CSS 4 with design tokens
- **Testing**: Vitest (unit/integration), Playwright (E2E)
- **Build**: TypeScript strict mode, ESLint

**Layer Architecture**: Presentation → Game → Service → Repository → Persistence

## Upgrade Notes

### First Installation
This is the first alpha release. No migration required.

**Guest Mode Setup**:
1. Visit application URL
2. Data automatically persists to localStorage
3. No authentication required

**localStorage Keys**:
- `scan-chan-pet-store`
- `scan-chan-game-store`
- `scan-chan-settings-store`
- `scan-chan-inventory-store`

### Future Upgrades
Guest mode data will migrate to Arashu mode when authentication is implemented.
